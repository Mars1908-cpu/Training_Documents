	- Package Management :
	
		- Packages are applications/software that are running on Linux Machines.
		
		- Package Manager: is used to Install/Uninstall/Update the Package.
		
			- On Linux OS :
			
				The Linux Packages managers are based on the Linux Distributions
				
				
				yum 			==> Centos/RHEL Distributions
				
				apt/apt-get 	==> Debian/Ubuntu Distributions
				
				dnf 			==> Fedora Distributions
				
				dnf 			==> Amazon Linux(Latest Version)
				
				yum 			==> Amazon Linux(Previous Version)
				
				
		- On Linux Machines, Package Managers are executed only as a root(admin) user.
		
			
			Login to Linux Machine as a ubuntu user & elevate the access to root user.
			
				$ sudo -i 		# Elevate the Access to root user 
				
				$ exit 
				
				
		Linux Commands :
		
			clear 				# Clear the Screen

			$ sudo -i 			# Elevate the Access to root user 
			
			$ exit 
				
			apt update -y		# Update the Default packages
			
			apt install git -y 
			
			apt remove git -y 
			
			apt upgrade git -y 
			
		How to Install the Packages - Best Practise ?

			- Type of Package ?
			
				-> Stand-alone Application Package 
						
						- git 
				
				-> Service Package 
				
						- Web Application Server - tomcat/nginx/httpd 
						
							tomcat 			=> 8080
							nginx/httpd  	=> 80 
							jenkins			=> 8080
							
							Access the Service Package use: <public_ip_address>:<port_number>
							
							As a best practise always refer to the officical documentation of the tools
						
						
			- Installation Process :				Eg.: 	To Install Tomcat
			
				- Install the Pre-requisites 				Install Java
			
				- Install Actual Package 					Install Tomcat 
				
				- Post Installation Activities				Start/Stop Tomcat Service 
		
		- Install Stand-alone Package :::
			
			Eg.: To Install Git on Linux Machine : https://git-scm.com/install/
			
				$ apt install git -y 
				
				$ git --version 			# To verify the installed version
				
				apt remove git -y 			# To Uninstall 
				
				apt upgrade git -y 			# To Upgrade
				
		- Install Service Package :::
		
				apt install nginx -y 
				
				apt install docker.io -y 
				
				
				Sample: Install git,docker,jenkins,kubernetes,ansible,terraform,prometheus/grafana

				- Start / Stop / Enable / Status / restart / disable
				
				systemctl status nginx 
				
				systemctl start nginx
				
				systemctl stop nginx
				
				systemctl enable nginx		# Used to define the service as as start up service
		
				systemctl disable nginx		# Used to remove the service from the start-up task list
				
				systemctl restart nginx

- Linux	: File Management :::

			Navigate to Directories/Folders :::

				- pwd				# Get the Present Working Directory

				- cd				# Change Directory
				
				- cd <dir_name> 	# Change to a specific directory				
				
				- cd /				# This command will take to the System root directory			
			
				- cd ~				# This command will take to the home directory(Default Directory) of the specific linux user
									Eg.: 	ubuntu 	===> /home/ubuntu 
		
											root	===> /root
									
				- cd .. 			# Back to previous directory
	
				- ls 				# Is to List the files and Directories 
				
				- ls -a 			# Is to List all the files and Directories(Including the hidden components) 	

				- ll				# Long List 
				
				
	- File Operations :
	
		Create/Open/Read/Edit/Delete/Update/Copy/Rename
		
		touch file1.txt							# To Create an empty file
		
		echo "Hello Everyone!" 					# To print a message to the terminal 
		
		echo "Hello Everyone!" >> file1.txt 	# To Create/Add More records to a file 
		
		echo "Hello Everyone!" >  file1.txt 	# To Remove all the existing records and Create a records to a file
		
		cat file1.txt 							# Read the File content 
		
		mv file1.txt tempfile.txt 				# To Rename a file / Directory
		
		mv dir1 dir2 							# To Rename a file / Directory
		
		cp tempfile1 dir1 						# To Copy a file to a target directory
		
		mv file1.txt targetdir1					# To Move a file to a target directory
